
Le pattern Facade fournit une interface unifiée à un ensemble d'interfaces dans un sous-système.

Le pattern Facade offre une interface simplifiée pour un ensemble complexe de fonctionnalités (ou de classes).

Elle fournit une interface simple et unifiée qui permet à un client d'interagir avec le système sans avoir à comprendre tous les détails internes.

CAS D'UTILISATIONS:

==>     Utilisation fréquente avec la creation des bibliothèques ou des frameworks : Il est couramment utilisé pour simplifier l'utilisation de bibliothèques ou de frameworks complexes
en offrant une interface utilisateur plus conviviale.
